# ⚒ XGEN MAKER

**한 문장으로 코드를 고칩니다 — 코드베이스 지식그래프에 발을 딛고.**

평소 말로 요청하면, MAKER가 코드의 *어디를* 고칠지 찾고, 최신 브랜치를 따서,
코딩 에이전트에게 구현시키고, **통과할 때까지 스스로 테스트하고 고친 뒤**,
사람이 볼 수 있게 병합 요청까지 준비합니다.

## 왜 만들었나

코딩 에이전트는 코드를 *쓰는* 건 잘하는데 *어디에 쓸지*는 모릅니다. 저장소가 여러 개인
플랫폼에서 "로그인 버그 고쳐줘"는 곧 — 수천 개 파일에서 맞는 자리를 찾고, 그걸 쓰는 쪽이
누군지 알고, 그들을 안 깨뜨리고, 팀의 브랜치·리뷰 규칙을 지키는 일입니다.

MAKER는 에이전트 **앞에 코드 지식그래프**를, **뒤에 수렴 루프**를 답니다.

```
요청: "온톨로지 그래프가 재빌드 후 안 바뀌는 버그 고쳐줘"

  ├─ 의도 분류        버그 / 기능 / 리팩터 / 질문
  ├─ 그래프에서 착지   → 저장소:경로:줄  (심볼 정확도 98%)
  ├─ 의존자 파악      이걸 바꾸면 누가 깨지는가
  ├─ 최신화 + 브랜치   이름 규칙 · 보호 브랜치 가드
  ├─ 에이전트 구현     그래프 맥락 + 실제 소스 발췌를 쥐여주고
  ├─ 검증            문법 · 테스트 · 샌드박스 격리
  ├─ 품질 평가        기준 미달이면 되먹여 재시도
  └─ 병합 요청 준비    ← 여기서 멈춥니다. 의도적으로.

사람: 리뷰 · 머지 · 배포
```

**MAKER는 배포하지 않습니다.** 병합 요청까지 준비하고 관측만 합니다. 머지와 릴리즈는 사람 몫입니다.

## 기능

| | |
|---|---|
| 🕸 **코드 지식그래프** | 파일·클래스·함수·엔드포인트·라우트를 AST 수준으로, `contains`·`imports`·`calls` 관계까지. 저장소를 넘나들며 연결되고, 전체 재빌드 없이 증분으로 최신 유지 |
| 🎯 **근거 있는 착지** | 자연어 요청 → 고칠 정확한 `저장소:경로:줄`. 여기에 **그 코드를 쓰는 곳**까지 함께 넘겨, 에이전트가 호출부를 깨뜨리지 않게 |
| 🔁 **수렴 루프** | 구현 → 샌드박스·테스트·회귀 → 품질 평가 → 실패를 되먹여 재시도. 통과하거나, 정직하게 포기 |
| 🛡 **구조적 안전장치** | 보호 브랜치 불가침, 브랜치 이름 규칙 강제, 인프라 파일 거부, 병합 요청까지만, 한 번에 되돌리기 |
| 📊 **웹 대시보드** | 실시간 실행 로그, 파고드는 그래프 뷰어, 이력·재실행·되돌리기, 테스트 기록, 화면 회귀 검증, 건강도 지표 |
| 🖥 **세 가지 표면** | CLI · 웹 대시보드 · MCP 서버 — 셋 다 같은 엔진 |

## 시작하기

```bash
pip install -e .                                   # maker 명령이 생깁니다

cp .env.example .env                               # 자기 토큰을 채웁니다
cp maker.config.example.json maker.config.json     # 저장소 경로를 매핑합니다

maker login                                        # Claude CLI 세션을 감지
maker doctor --config maker.config.json            # 모든 능력이 실제로 되는지 검증
```

> **설정 없이는 아무것도 안 됩니다 — 의도한 것입니다.** 이 저장소에는 예시값만 있습니다.
> 실제 주소·토큰·저장소 경로는 `.env`와 `maker.config.json`에만 두며, 둘 다 git에서 제외됩니다.

### 그래프 만들기

```bash
maker kg build --repo "core=/경로/core" --repo "web=/경로/web::apps/web/src" --out kg
maker kg merge kg/*.repo.json --out kg/merged.json
maker kg enrich --kg kg/merged.json        # 선택: 의미 요약 채우기
```

### 요청 실행

```bash
maker run "로그인 리다이렉트 버그 고쳐줘" --config maker.config.json                    # 분석만
maker run "로그인 리다이렉트 버그 고쳐줘" --config maker.config.json --mode observe     # + 브랜치·커밋
maker run "로그인 리다이렉트 버그 고쳐줘" --config maker.config.json --mode act         # + 푸시·MR
```

### 또는 대시보드

```bash
maker web --config maker.config.json        # http://127.0.0.1:8760
```

## 실행 모드

의도에 비례해 권한이 열립니다. 기본값은 아무것도 건드리지 않습니다.

| 모드 | 저장소 | 원격 | 쓰임 |
|---|---|---|---|
| `plan` *(기본)* | 손대지 않음 | — | 탐색, 질문에 답 받기 |
| `observe` | 로컬 브랜치 + 커밋 | — | 내 컴퓨터를 벗어나기 전에 변경 확인 |
| `act` | 로컬 브랜치 + 커밋 | 푸시 + 병합 요청 | 팀에 넘기기 |

모르는 모드 값은 **거부**합니다. 조용히 쓰기 권한으로 올라가지 않습니다.

## 대시보드

```bash
maker web --config maker.config.json
```

| 탭 | 하는 일 |
|---|---|
| **실행** | 단계별 실시간 로그, 착지한 코드, 실제로 에이전트를 죽이는 중지 버튼 |
| **파이프라인** | 25단계 전부와 각 단계를 켜고 끄는 설정 — 그 자리에서 변경 |
| **지식그래프** | 저장소 관계도 → 저장소 내부 → 노드를 누르면 실제 소스와 AI 요약. 메모를 달면 재빌드해도 남습니다 |
| **작업 이력** | 세션별 단계 타임라인, 이어서 실행, 한 번에 되돌리기 |
| **테스트** | 실행별 검증 기록 — 샌드박스, 검사, 품질 점수와 그 근거 |
| **화면 검증** | 화면을 캡처해 기준으로 저장하고, 이후 변경을 픽셀 단위로 비교 |
| **건강도** | 저장소별 최신 여부, 무결성, 심볼 정확도 — 주장이 아니라 측정값 |

대시보드에는 **인증이 없습니다.** 그래서 명시적으로 동의하지 않으면 외부 주소에 바인딩을
거부합니다. 포트에 닿는 사람은 저장된 자격으로 행동할 수 있으니, 외부에 열려면 인증
프록시 뒤에 두세요.

## 그래프를 최신으로 유지하기

그래프가 오래되면 에이전트가 엉뚱한 파일로 갑니다. 그래서 최신성은 일급 관심사입니다.

```bash
maker kg sync              # 로컬에서 바뀐 것만 다시 추출
maker kg refresh           # 원격을 받아오고, 안전할 때만 fast-forward한 뒤 sync
maker kg hook --install    # git 훅: 커밋·머지·체크아웃 때 자동 갱신
```

`refresh`는 일부러 보수적입니다. **fetch는 항상** 하고(워킹트리를 건드리지 않습니다),
fast-forward는 ① 워킹트리가 깨끗하고 ② 업스트림이 있고 ③ 갈라지지 않았을 때만 합니다.
아니면 건너뛰고 **이유를 알려줍니다.** checkout·stash·rebase·force는 하지 않습니다.

매일 한 번 걸어두면 아무도 신경 쓰지 않아도 그래프가 최신을 유지합니다.

## 구조

```
                    ┌──────────── 표면 ────────────┐
                    │   CLI      웹(SSE)      MCP  │
                    └───────────────┬──────────────┘
                                    ▼
                           MakerLoop (파이프라인)
                                    │
  ┌──────────────┬──────────────┬───┴──────┬──────────────┬──────────────┐
  ▼              ▼              ▼          ▼              ▼              ▼
의도 분류      지식그래프      git 작업    코딩 에이전트     검증         병합 요청
             │             브랜치·커밋   (하위 프로세스)  테스트·샌드박스     초안
             ▼                                          ·품질평가
   build · sync · refresh · overlay · search · impact
```

- `xgen_maker/kg/` — 그래프 구축, 증분 동기화, 안전 최신화, 검색, 사람의 편집(overlay)
- `xgen_maker/loop/` — 파이프라인: 의도 → 착지 → 브랜치 → 구현 → 검증 → 평가 → MR
- `xgen_maker/web.py` — 대시보드 (표준 `http.server`, SSE, 단일 파일 UI)
- `xgen_maker/mcp_server.py` — 그래프와 플래너를 다른 에이전트에게 노출
- `scripts/` — 운영 도구 (정기 최신화, 터널 라우팅)

## 요구사항

- **Python 3.12+**, `git`
- **표준 라이브러리 우선.** 그래프·루프·대시보드 핵심은 서드파티 런타임 의존성이 없습니다.
  선택 기능만 추가로 필요합니다 — `Pillow`(픽셀 비교), `npx` 경유 Playwright(화면 캡처),
  LLM 제공자(의미 요약·품질 평가·질의 확장).
- 구현 단계를 맡을 코딩 에이전트 CLI — 기본은 Claude CLI, `agent_cmd`로 임의 명령 지정 가능.

## 안전 모델

MAKER는 운영에서 **심심하도록** 설계했습니다.

- **배포하지 않습니다.** 파이프라인은 병합 요청에서 멈춥니다.
- **보호 브랜치**는 생성·커밋·푸시가 모두 막힙니다.
- **인프라 파일**(Dockerfile, CI 정의, 차트)은 거부합니다. 소스 코드만 고칩니다.
- **권한 확인은 쓰기 전에** 합니다. 쓰고 나서가 아니라.
- **모든 과정이 세션별로 기록**되고 한 명령으로 되돌릴 수 있습니다.
- **실패를 정직하게 보고합니다.** 건너뛴 테스트는 건너뛰었다고, 미검증 회귀는 미검증이라고
  말합니다. 아닌 걸 초록으로 칠하지 않습니다.
